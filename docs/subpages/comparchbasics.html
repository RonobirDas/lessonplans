Intro to Verilog

First, if you’d like to make a simple testbench, I’ve covered that in a previous link found here Link

Now to learn verilog you must get two books, or rather two sets of books.

Books

Set 1
Verilog HDL (2nd Edition)
A Verilog HDL Primer, Third Edition

Set 2
Verilog Styles for Synthesis of Digital Systems
Verilog HDL Synthesis, A Practical Primer

At the very least, pick one book from Set 1 and one book from Set 2.

First and foremost the best class to follow for an intro to verilog is the EECS150 Course at UCBerkeley. This is a link to the Fall 2013 Class

For a quicker approach you may consider: CSE141L at UCSD

Finally a note on Synthesizable Verilog. If you plan to implement your design whether it be on an IC or an FPGA, you must use a subset of Verilog specifically for synthesis, as not all constructs are allowed.

Here is a list to follow including a cheat sheet:
Link to an MIT Page by Dr. Wentzlaf
VSynth Cheat Sheet
UTDallas Verilog HDL Tutorial with special attention to IC Design

System Verilog

SystemVerilog for Design Second Edition: A Guide to Using SystemVerilog for Hardware Design and Modeling - FROM STUART SUTHERLAND, one of the foremost experts in logic design.
SystemVerilog Assertions and Functional Coverage: Guide to Language, Methodology and Applications - Focuses on code coverage for testing in DUV systems.

Computer Architecture

If you’re attempting anything from architecture, such as SuperScalar or Branch Prediction, and the only thing you’ve read is a basic textbook, don’t even think of starting the design without the theory. Read the intermediate books from the first line that’s a link here. Read the book Computer Organization and Design, don’t even think about architecture until then.
